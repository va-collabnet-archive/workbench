Maintenance workflows

In this tab, maintenance worklists are used to send any concept into a translation workflow without the requirement of creating a new workset, e.g., in the case of an already translated concept incidentally found thay may nedd correction.

Many different workflows can be configured.

This tool is intended for single concepts only and therefore, worklists of only one concept will be generated.

You should introduce a name for your maintenance worklist and select a business process from the Workflow Business Process file. Then click “Create new Worklist”. The list of available workflows is configurable for each implementation and can grow according to needs:

  • NOTE: When “MaintenanceWorkflow.bp” is chosen, only translator and reviewer are involved: the translator corrects the concept, sends it to the reviewer and then, the concept enters directly to the Archival queue upon reviewer’s approval.

    When “IsolatedEdit.bp” is chosen, any person with permission may edit, correct and submit the concept to the Archival queue

    In this example, when “Translation Workflow.bp” is selected, the concept to be changed will follow the standard translation workflow, but other business processes may be configured according to the users needs.

The maintenance worklist automatically generated is displayed on the left panel:

Now you must specify the concept that will be corrected. Go to the Taxonomy tab and right click on the concept you will want to correct:

A window showing the Project name and the name of the maintenance worklist is opened. Click “>” button to assign the new worklist:

A confirmation message is shown:

Then, a new window is displayed, asking you to select the roles that will take part in the process (A) and the execution options (B):

(B)

If you select to execute the process immediately, the translation interface is opened and you may change the selected concept:

Once you have made the change and the correction process is complete, you may end your correction by clicking “Approve” button.

A message indicating that item translation is complete and that it has been sent to the Archival queue of the Release Manager es displayed.

The screen goes back to the Workset details pane upon clicking OK and the maintenance process is completed.

NOTE: Maintenance Workset, partitions and worklists are automatically generated for each project. These are not meant to be maintained by the user; concepts will be added to this component as needed. User changes are disabled for this workset:

Export function

This function allows to export refsets, worksets, partitions and worklists in release format 1 (RF1) and release format 2 (RF2).

Clicking one of the upper buttons will allow you to select the desired release format. Click the upper button on the right to select the folder where your file will be placed, while the lower button on the right allows selecting a file to log results.

Drag and drop the refset concept to the Refset concept line.

You may also select the file format (.txt or .csv):

Upon clicking Export button, the refset is exported according to your specifications and a message appears as shown below. Also, the Results window will display a summary description of the exported document.

In the example shown, you specified that a document with RF2 format would be exported. Two tables were generated as .txt files: TEST 2 (worklist)-ET 20101116170445_SCTID.txt and TEST 2 (worklist)-ET 20101116170445_UUID.txt.

The SCTID.txt file will show the following fields:
  • an internal UUID, the effective time (date when the last component version was generated, component status for RF2 (1 = active),
  • the component path,
  • workset UUID, and
  • SCTID (which in this case is the Concept ID because the selected worklist is a concept refset).

The UUID.txt document is similar, except that the UUID is displayed as a component identifier instead of SCTID.

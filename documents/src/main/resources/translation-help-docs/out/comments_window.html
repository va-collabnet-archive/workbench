Comments / Issues window

Comments tab

This tab displays comments with date, time, author and text. Upon clicking “[A]dd comment” button, a new window is opened. Some comment options may be chosen according to the type of comment to be made and a box for free text writing is provided:

Upon clicking OK button, the window is closed and the comment is displayed in a list under “Comment type”, ordered by date, time, author and comment text. A red number in brackets is shown in the “Comments” tab indicating the number of comments, for a given description:

Comments may be ordered by ascendent or descendent order, by clicking on the sorter:

Comments may also be deleted by right-clicking on a selected comment:

Issues tab

When issue repositories have not been configured by the project manager this tab is inactive.

When the project manager has specified the available repositories the word “Inactive” in brackets disappears when the user has specified his/her own username and password to enable issue posting in collabmet site.

In order to do this, users must open the project manager, select the project they will be working with and open the “Issues” tab on the right panel:

Notice that as the project manager has already defined all the mandatory fields and specified the available repositories, the translation workflow users must only write their credentials in the corresponding windows and then click save:

Back in the translation interface, selecting the Issues tab will display the Issues Interface. The size of this window may be changed according to the users preferences by scrolling the lateral bars to both sides or by clicking the expand/reduce icons:

A new issue window is opened upon clicking “Create Issue” button; the user must enter an Issue title and a description of the problem reported:

A message confirming that the issue has been posted is displayed upon clicking OK:

Go to https://csfe.aceworkspace.net > IHTSDO Translation Tooling Project > Tracker > Specific Translation project issues (EB) Summaries and you will find the posted issue in an artifact list. The issue title is displayed. You must select this link to view artifact's details:

When the issue has been posted, it is shown as a table in the Issues window of the translation interface, with Issue Title, Description, Status and the user who posted it:

Notice that there is an ‘Issue Comments’ window on the right. When any other user in the workflow has further comments to the issue, he/she may enter them in this window:

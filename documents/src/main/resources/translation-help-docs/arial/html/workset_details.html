Workset details

REMEMBER! A workset is a refset “frozen” in a point of time and associated to a project.

Creation if worksets

  • Open the worksets tab on the right panel and click “Create new workset”. A new window is displayed in order to enter a new workset name. (You will not be able to create two worksets with the same name, an alert message will be displayed).
  • Once you have entered a new workset name you should define the refset from which you will obtain the concepts to be translated. Once again, the source refset and any additional features may be displayed by clicking the respective tabs on the right panel.
  • In order to create a workset, you should follow these steps:
    • Define your source refset: Open the taxonomy tab on the left pane and select the refset containing the concepts to the translated. Drag and drop it into the Source Refset field in the right panel. Then click “save”:

  • You may also drag and drop refsets from the tree view in the left pane (e.g., if you had specified a linked refset you may drag and drop it into the source refset window from the linked refset folder.

  • Save your specifications. (If you have forgotten to save your source refset and try to synchronize without saving, an error message will de shown.)

  • Upon saving the source refset, the list and the number of refset members is displayed. However, if you open the Workset members tab, you will see that it is still empty:

    The list of workset members will only be shown after synchronizing the workset with the source refset.

    • Define your exclusion refsets (if any): Whenever you have a refset of concepts that will not be translated, you should specify them as an exclusion refset before synchronizing your refset. Any exclusion refset must be defined in the project. Otherwise, the refset will not be excluded.

    Then, you must synchronize your workset with the source refset in order to ensure they are identical.

    NOTE: A refset is mutable and does not belong to the Translation Process, but a Workset does. Therefore there is a need to keep one or more copies of a set of data consistent with the refset created for the translation process

    Click “Sync” button to synchronize your refset. A confirmation message is displayed:

    Once your workset has been synchronized, a message will be displayed, and if you open the Members tab, now tou will see that the workset members list is exactly as you specified in your source refset:

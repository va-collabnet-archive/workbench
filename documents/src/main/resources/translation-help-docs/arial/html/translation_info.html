Translation help window

This window is shown in a separate tab on the left pane for a better visualization. The help window is divided in three sections:

  • An upper Similarity section that shows similar translations. An enlarged view is displayed upon clicking “E[x]pand” button. The user may also choose to see FSNs, Preferred terms or both. In the expanded view, a Status column has been added. Whenever a displayed similarity match exists with a concept included in the current project, the status for that concept in the translation workflow is also listed:

If the user selects a similarity match by FSN, matching source and target FSN are sought. If the target language has not translated FSNs, no similarity matches will be displayed (A). If matching criteria are specified by preferred term the list of matches will be displayed (B). Finally, if “Both” button is selected, matching will be sought for FSNs and PTs. If the target language has no translated FSNs, the PTs will be shown twice (C). In case that target language has used source FSNs, these will be displayed in the target FSN rows (not shown):

When “Refine” box is ticked, a search option is opened, and any term is searched. In this window, the user may write any word he/she wants to find for similarity in the source language, and the results are displayed below:

  • A middle Translation memory section, which displays similar semantic structures that have already been translated, in order to maintain translation consistency:

  • A lower Linguistic Guidelines section, where applicable linguistic rules are displayed. Also, false cognates are presented in this section.

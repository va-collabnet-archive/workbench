Boolean Refset

An optional file containing all the boolean refsets in the terminology database, and their status and version information.

Filename: boolean.refset

Fields

Name Type Notes
refset uuid UUID
Definition The UUID of the refset that this record is a member of. record.
Coding See : refset uuid
member uuid UUID
Definition The UUID of the refset member.
Coding See : member uuid
status uuid UUID
Definition The UUID of the concept's status concept.
Coding See: Getting a Status UUID
component uuid UUID
Definition The UUID of the member's referenced component.
Coding See : component uuid
effective date Date
Definition The date this change became, or will become, effective, with a required granularity of seconds.
Coding See: Effective Date
path uuid UUID
Definition The UUID of the development path upon which this record is committed.
Coding See: PathUUID
boolean extension value Boolean
Definition A boolean value that extends a component.
The value is represented by a string "f" or "false" for false or "t" or "true" for true.
Coding 
blRefset.append("t");

RefSet Common Coding Notes

This is where coding task description which are used in more than one place go so as to avoid copy and paste in the main text file table descriptions.

Name Notes
refset uuid The refset uuid is usually obtained via getting a Concept UUID from the Concept enum within the org.dwfa.cement.RefsetAuxiliary
class, rather than from the Concept enum within the org.dwfa.cement.ArchitectonicAuxiliary
class.
  • See Getting a Concept UUID
  • Example: 
    conceptRefset.append(RefsetAuxiliary.Concept.CTV3_SUBJECT_TYPE.getUids().iterator().next().toString());
member uuid The member uuid has to be unique for the refset tuple.
  • See Generating a UUID
  • Example: 
    conceptRefset.append(Type5UuidFactory.get(memberTypeFivePrefix,conceptparts[readcodeIndex]).toString());
  • You may need to append a number of values to give you a unique string which will then guarantee a unique uuid for example: 
    intRefset.append(Type5UuidFactory.get(ReadStatics.READV3_REL_ID,
relationshipParts[objectReadCodeIndex]+relationshipParts[attributeReadCodeIndex]+relationshipParts[valueReadCodeIndex]).toString());
component uuid The component uuid is used in refset specs to specify the parent of the particular tuple by matching its member UUID.
If the component uuid is the refset uuid, then it is at the spec's top level (under the implicit OR). e.g. 
AND (component ID == refset UUID, member UUID == a)
CONCEPT IS MEMBER OF: refset 1 (component ID == a, member UUID = b)
CONCEPT IS MEMBER OF: refset 2 (component ID == a, member UUID = c)
  • See Generating a UUID
  • Example: 
    intRefset.append(Type5UuidFactory.get(ReadStatics.READV3_REL_ID,
relationshipParts[objectReadCodeIndex]+relationshipParts[attributeReadCodeIndex]+relationshipParts[valueReadCodeIndex]).toString());
